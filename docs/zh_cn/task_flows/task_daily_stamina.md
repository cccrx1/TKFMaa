# 消耗体力（DailyStamina）

入口节点：`DailyStaminaStart`；实现涉及 `daily_stamina*.json`、`regular_activity.json`、`assets/stamina_activities.yaml`、`tools/build_stamina_activities.py` 和 `agent/stamina.py`。原体力配置、生成方案、逻辑核查与待办已合并到本文。

## 前置状态

- 游戏位于主界面，目标活动和关卡已解锁。
- 在“体力消耗关卡”中确认目标；活动名称、路线、关卡文本和体力消耗以 `assets/stamina_activities.yaml` 为源配置。
- 确认“扫荡不足时手动补刷”“使用体力药”和“购买体力”开关。后两项会消耗资源，默认关闭。

## 主流程

1. 从主界面进入出征，由 `DailyStaminaPresetRouteStart` 选择限时活动、日常政务或自动通关路线。
2. 按活动名称进入内部页面；普通列表寻找指定关卡，自动通关路线通过 `RegularActivityStart` 推进带 `NEW` 标记的关卡。
3. 进入关卡详情后由 `DailyStaminaCompare` OCR 当前体力，并与关卡消耗及保留值比较。
4. 体力足够时优先打开军团扫荡；扫荡不可用且开关启用时改为手动战斗。
5. 关闭扫荡或战斗结算，重新判断是否继续；体力不足、目标不存在或路线完成时返回主界面。

## 页面识别

| 页面状态       | 识别目标                              | 识别方式                    | ROI/素材                       |
| -------------- | ------------------------------------- | --------------------------- | ------------------------------ |
| 活动区与活动卡 | 限时活动/目前活动及活动名称           | OCR                         | 由 YAML 模板生成 Override      |
| 关卡列表       | 目标关卡、列表 marker 或第一页 marker | OCR                         | 由活动配置提供 ROI             |
| 自动通关目标   | 金色候选区域内的完整 `NEW`            | ColorMatch + 动态 ROI OCR   | `regular_activity.json`        |
| 关卡详情       | 军团扫荡、体力值                      | OCR + `DailyStaminaCompare` | 体力 ROI 默认 `[500,0,180,45]` |
| 战斗与结算     | `WAVE`/`TURN`、返回地图、完成/跳过    | OCR                         | 战斗及结算区域                 |

## 异常与分支

- 体力不足时仅按启用选项使用一次体力药或购买一次体力；选项关闭、道具不可用或已补充过时取消弹窗并退出。
- 普通列表有限次上下扫描仍找不到关卡时安全返回，不无限滑动。
- 自动通关只点击经过颜色候选和局部 OCR 双重确认的 `NEW`；当前页没有目标时仅在确认 `NEXT` 已解锁后翻页。
- 已知待复核风险：扫荡弹窗较晚出现 Loading 时可能提前回退；扫荡开始后未知网络/活动结束弹窗缺少专用 `on_error`；复刻地图边界仍依赖 `max_hit`；活动选择页只覆盖当前可见槽位。

## 完成状态

- 成功条件：完成目标扫荡/自动通关，或达到体力、道具、关卡解锁等停止条件。
- 安全退出位置：游戏主界面；未知补充体力弹窗优先取消，不进行资源消耗。
