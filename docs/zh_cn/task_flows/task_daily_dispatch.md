# 每日派遣（DailyDispatch）

入口节点：`DailyDispatchStart`；实现文件：`assets/resource/pipeline/daily_dispatch.json`。

## 前置状态

- 游戏位于主界面，地城派遣功能已解锁。
- 确认“派遣地点优先级”和“派遣时间优先级”；可用角色数量可能不足。

## 主流程

1. 打开左侧抽屉，根据栏位状态进入地城派遣总览。
2. 优先打开“派遣完成”栏位，收回队伍并关闭归来弹窗。
3. 打开“闲置中”栏位，按配置顺序选择地点和派遣时长。
4. 先尝试条件推荐，再尝试人员补充；仍未满队时手动填充队伍。
5. 队伍满足出发条件后派出，继续扫描其他栏位；全部忙碌后返回主界面。

## 页面识别

| 页面状态   | 识别目标                          | 识别方式            | ROI/素材         |
| ---------- | --------------------------------- | ------------------- | ---------------- |
| 抽屉栏位   | 派遣完成、闲置中、派遣队1与倒计时 | OCR/And             | 左侧抽屉栏位区   |
| 派遣总览   | 地城派遣                          | OCR                 | 页面标题区       |
| 归来弹窗   | 队伍归来/隊伍歸來                 | OCR                 | 弹窗内容区       |
| 地点与时间 | 地点名称和对应时间按钮            | OCR + target offset | 选择页面         |
| 编队完成   | 出发按钮和队伍槽蓝绿色            | And + ColorMatch    | 编队与底部按钮区 |

## 异常与分支

- 条件推荐失败后依次使用人员补充和手动填队；仍不满队则放弃该栏位并返回总览。
- 所有地点均无可选时间时返回总览，不重复进入选择页面。
- 总览既无完成队伍也无闲置队伍时视为全部忙碌并结束。
- 地点/时间优先级通过 `pipeline_override` 调整 `next` 顺序，目标节点必须预先存在。

## 完成状态

- 成功条件：已收回全部完成队伍，并处理所有可派遣的闲置队伍。
- 安全退出位置：游戏主界面；编队无法满足时允许保留闲置队伍。
