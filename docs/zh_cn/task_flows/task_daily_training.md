# 调教（DailyTraining）

入口节点：`DailyTrainingStart`；实现文件：`assets/resource/pipeline/daily_training.json`、`agent/training.py`。

## 前置状态

- 游戏位于主界面，调教功能已解锁。
- 确认调教目标、道具范围和珍贵道具开关；“指定次数”模式还需填写 1-100 次。
- 当前界面说明明确标注该功能尚未充分测试，使用珍贵道具前必须实机确认。

## 主流程

1. 主界面调教次数不为零时进入调教，否则直接结束。
2. 打开筛选面板，将调教进度切换为“可调教”，确认后进入带 `NEXT` 的房间。
3. 按配置优先选择通用或活动道具栏，由 Agent 读取库存并选择可用道具。
4. 按指定次数点击，或长按消耗全部点数。
5. 道具/点数不足或没有可调教房间时返回主界面。

## 页面识别

| 页面状态   | 识别目标                                 | 识别方式     | ROI/素材                  |
| ---------- | ---------------------------------------- | ------------ | ------------------------- |
| 主界面次数 | `/50` 或 `0/50`                          | OCR          | 调教按钮计数区            |
| 筛选面板   | 筛选排序、调教进度、可调教               | OCR          | 筛选面板                  |
| 可调教房间 | `NEXT`                                   | OCR          | 房间列表                  |
| 道具选择   | 通用、活动及固定槽位库存                 | OCR + Custom | `DailyTrainingChooseItem` |
| 终止弹窗   | 调教点不足、禁忌果实、补充能量或道具不足 | OCR          | 弹窗内容区                |

## 异常与分支

- 无 `NEXT` 房间时从列表返回；首选道具栏无可用道具时尝试另一栏。
- Agent 默认跳过 +2000/+2400 珍贵道具，只有开关启用时允许选择。
- 库存 OCR 失败目前会把该槽位视为“可能有道具”，存在误选风险，需重点实测。
- 点数或道具不足时取消弹窗，不自动购买或补充资源。

## 完成状态

- 成功条件：完成指定次数/耗尽允许使用的点数，或没有可用房间与道具。
- 安全退出位置：游戏主界面。
